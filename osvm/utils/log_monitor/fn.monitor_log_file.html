Create a log file monitor that tails the file (like tail -f)

§Signal Handling

This function runs an infinite loop monitoring log files. To ensure graceful shutdown on Ctrl+C (SIGINT) or termination signals, wrap the call in signal handling:

ⓘ
use tokio::select;
use tokio::signal;

// Usage example:
tokio::select! {
    result = monitor_log_file(path, config, callback) => {
        // Monitoring ended naturally
        result?;
    }
    _ = signal::ctrl_c() => {
        eprintln!("Log monitoring interrupted by user");
        // Cleanup happens automatically via Drop trait
    }
}

Or use a signal handler for graceful termination of any child processes spawned during the monitoring process.

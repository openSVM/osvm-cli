Node Deployment

▓▓ SSH Deployment

Deploy validators and RPC nodes to remote hosts via SSH:

osvm user@host --svm sonic --node-type validator --network mainnet

Deploy Sonic validator to mainnet

osvm user@host --svm solana,eclipse --node-type validator --network devnet

Deploy multiple SVMs to devnet

osvm user@host --svm sonic --node-type rpc --network testnet

Deploy RPC node to testnet

▓▓ Node Management

osvm nodes list

List all nodes with status and metrics

osvm nodes dashboard

Real-time monitoring dashboard with TPS, CPU, memory

osvm nodes status node-123

Check specific node health and status

osvm nodes logs node-123 --follow

Stream node logs in real-time

osvm nodes restart node-123

Restart node with zero-downtime (if hot-swap enabled)

▓▓ Deployment Configuration

DeploymentConfig {
    svm_type: "sonic",
    node_type: "validator",
    network: "mainnet",
    hot_swap_enabled: true,
    metrics_config: {
        port: 8899,
        enabled: true
    }
}

▓▓ Hot Swap Updates (Zero-Downtime)

OSVM supports zero-downtime updates with hot-swap technology:

  • Blue-green deployment: Deploy new version alongside old
  • State preservation: Transfer state from blue to green
  • Automatic health checks: Verify green before switching
  • Instant rollback: Return to blue if issues detected
  • <1ms downtime: Seamless traffic switching

▓▓ System Optimization

Automatic system tuning during deployment:

  • File descriptor limits: Set to 1,000,000+
  • Network parameters: TCP buffer tuning
  • Memory management: Swappiness, transparent huge pages
  • CPU affinity: Pin processes to cores
  • I/O scheduler: Deadline or noop for SSDs
  • Kernel parameters: Solana-optimized sysctl values
